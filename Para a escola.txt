1. Reestruturação do Acesso (Login e Perfis)
Atualmente, o seu LoginPage.tsx já salva o perfil do usuário: localStorage.setItem('user_perfil', response.data.perfil);. Vamos utilizar esse dado para decidir o que cada um vê.

Estratégia de Direcionamento:
Perfil 'Admin' ou 'Secretaria': Direciona para o Dashboard atual com todas as funções de gestão (Alunos, Professores, Financeiro).

Perfil 'Professor': Direciona para um Dashboard Simplificado focado em: Chamada Digital e Lançamento de Notas.

Perfil 'Aluno/Responsável': Direciona para o Portal do Aluno (Visualização de Notas, Frequência e Boletos).

2. Implementação do "Coração" do Sistema: Chamada Digital
Como você mencionou, a lógica de PercentualFrequencia no C# precisa de uma interface. Criaremos o componente ChamadaDigital.tsx.

O Fluxo será:

Professor seleciona a Turma.

O sistema lista os alunos daquela turma.

O professor marca Presença/Falta (Checkbox ou Switch).

O Backend C# calcula automaticamente o percentual de faltas e atualiza o status do aluno.

3. Portal do Aluno (Área dos Pais)
Para criar o acesso via Matrícula (RA), faremos o seguinte:

No Cadastro (AlunoForm.tsx): Adicionaremos um campo de Senha Inicial. O login do aluno será o registroAcademico (RA) e essa senha.

Na Tela de Login: Se o sistema detectar que o login foi feito com um RA, ele redireciona para a rota /portal-aluno em vez de /dashboard.

Funcionalidades do Portal do Aluno:
Meu Boletim: Consome a função que já temos de getNotasPorAluno.

Frequência: Gráfico simples de presença.

Financeiro: Visualização das mensalidades geradas na FinanceiroLista.tsx que estão como "Pendentes".

4. Ajuste nos Ícones da Tela de Login
Você perguntou sobre os ícones de "Biblioteca", "Alunos" e "Secretaria" no rodapé do login. Eles podem servir como Landing Pages informativas ou filtros de acesso rápido:

Secretaria: Link para os pais solicitarem documentos online.

Alunos: Atalho direto para a tela de login exclusiva do aluno.

Biblioteca: Consulta ao acervo (Módulo futuro).
